## 功能描述

AgoraMarket 質押功能允許用戶將資產鎖定以獲取利息收益。核心流程包含：

1. 查看質押頁面（利率、可質押餘額）
2. 發起質押（輸入金額 → 確認 → 成功）
3. 解質押（申請解鎖 → 確認 → 結算）
4. 查看歷史記錄（進行中 / 已結算 / 已解質押）

此功能曾多次回報異常，根本原因在於缺乏明確 AC，導致開發者與報告者對「修好」的定義不一致。

## 驗收標準（Acceptance Criteria）

### AC-1：質押頁面載入

- 質押頁面可正常開啟，無白屏或 JS error
- 顯示質押利率（數值非 `0`、非 `N/A`、非空白）
- 頁面顯示用戶當前可質押金額

### AC-2：質押發起流程

- 輸入有效質押金額後，點擊確認按鈕
- 系統顯示確認對話框或 loading 狀態
- 操作成功後：
  - 餘額扣款數值正確
  - 質押狀態變更為「質押中」或「進行中」
- 全程不出現 500 error 或 "Something went wrong"

### AC-3：解質押流程

- 解質押按鈕可見且可點擊（非 disabled）
- 點擊後顯示確認對話框
- 確認後質押狀態更新為「已解質押」或「結算中」
- 不出現未預期的錯誤

### AC-4：質押歷史記錄

- 歷史記錄頁面可正常開啟
- 能看到以下三種狀態的記錄（若有）：
  - 進行中（Active）
  - 已結算（Settled）
  - 已解質押（Unstaked）
- 各記錄顯示完整欄位：金額、開始日期、利率、狀態

### AC-5：金額顯示精度

- 質押金額與後端 API 回傳的小數位一致
- 不出現浮點精度問題（例如 `0.10000000001` 或 `0.0999999999`）

## 已知邊界情況（Edge Cases）

| 情況 | 預期行為 |
|------|----------|
| 輸入金額超過可用餘額 | 顯示明確錯誤提示，不執行扣款 |
| 輸入金額為 0 或負數 | 確認按鈕應 disabled 或顯示驗證錯誤 |
| 質押期間餘額不足手續費 | 系統提示，不靜默失敗 |
| 網路中斷後重試 | 不雙重扣款；冪等處理 |
| Telegram Mini App 環境 | 與 Web 瀏覽器行為一致（環境差異為高風險點） |
| 小數金額（例如 0.001） | 精度保持，不截斷或四捨五入為 0 |
