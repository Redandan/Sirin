Integrate the LanceDB vector memory into the Telegram message handling pipeline.

Goal: When a Telegram message arrives, search for relevant past context from memory and include it in the LLM prompt. After generating a reply, store the exchange in memory.

Current state:
- `src/memory.rs` has `add_to_memory(text, vector)` and `search_memory(query_vector)` fully implemented
- These functions are never called from telegram.rs or followup.rs
- No Tauri IPC commands expose memory operations
- Memory database stored at `data/sirin_memory`

Key challenge: memory.rs uses LanceDB with vector embeddings, but there is currently no embedding model integration. The `vector` parameter is a `Vec<f32>` that must be generated by an embedding model.

Implementation plan:
1. **Check memory.rs API**: Read src/memory.rs to understand the function signatures
2. **Add simple text-only memory**: Since there's no embedding model, add a simpler text search fallback — store recent conversation turns as JSONL in `data/sirin_context.jsonl`, then do keyword match retrieval
3. **Wire into telegram.rs**: 
   - After receiving a message, load last 5 context entries matching keywords from the message
   - Append a `Recent context:` block to the LLM prompt  
   - After sending a reply, append `{user: text, assistant: reply}` to context log
4. **Add Tauri commands**: Expose `clear_memory` and `search_memory_text` commands in main.rs

Files to modify: src/memory.rs, src/telegram.rs, src/main.rs

Read all three files before making any changes. Run `cargo check` after each modification.
